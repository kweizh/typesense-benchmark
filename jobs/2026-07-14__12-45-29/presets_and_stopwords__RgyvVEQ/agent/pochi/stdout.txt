# Typesense Search Presets and Stopwords ## Background You are building a small search backend on top of https://typesense.org, a fast, typo-tolerant in-memory search engine. The Typesense server binary is already installed in the environment at `/usr/local/bin/typesense-server`, but it is **not running**. Two of Typesense's operational features are central to this task: * **Presets**: a named, server-side bundle of search parameters that a client can reference by name at query time (via the `preset` parameter) instead of resending every parameter on each request. * **Stopwords sets**: a named list of keywords that Typesense removes from the *search query* (not from indexed documents) before matching. Your job is to stand up a Typesense instance, index a fixed catalogue of books, register a reusable preset and a stopwords set, and expose a small command-line tool that runs searches. - `--q "<query>"`: hits `GET /collections/library/documents/search` with **only** `q` and `preset=library_default`. No `query_by`/`sort_by`/`stopwords` are resent.
- `--q "<query>" --explicit`: hits the same endpoint but passes `query_by=title,author`, `sort_by=points:desc`, `stopwords=en_stopwords` directly (the preset content is mirrored in `PRESET_VALUE` so the two modes stay in lockstep).
- Both search modes print exactly `{"found": <int>, "hits": ["<id>", "<id>", ...]}` to stdout, with IDs taken from each hit's `document.id` in the order Typesense returns them.
