Each searchable record is a *customer* document whose data is deeply nested: a customer has an array of `orders`, every order has an array of `line_items`, and every line item carries a nested `attributes` object. This is a three-level nesting shape (`orders` -> `line_items[]` -> `attributes{}`), where the intermediate levels are arrays of objects. Typesense can index and query nested objects, but only when the collection explicitly enables nested fields and the sub-fields buried inside arrays of objects are typed correctly.
