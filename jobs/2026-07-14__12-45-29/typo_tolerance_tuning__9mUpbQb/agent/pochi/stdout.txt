### 3. `search.py` — Fixed typo-tolerance configuration
Located at `/home/user/typo-tuning/search.py`. Uses the official `typesense` Python SDK. The same fixed configuration is applied to every query — no branching on query text. The settings that satisfy **all five rules simultaneously** are:

| Parameter | Value | Purpose |
|---|---|---|
| `query_by` | `name,brand` | Required field order |
| `prefix` | `False` | Disable prefix matching |
| `num_typos` | `"2,0"` | **Rule 1** — 2 typos on `name`, 0 on `brand` |
| `min_len_1typo` | `4` | **Rule 2** — length-3 tokens never typo-corrected; length-4 may have 1 typo |
| `min_len_2typo` | `6` | **Rule 1** — 2 typos allowed on six-letter names like `camera` |
| `typo_tokens_threshold` | `1` | **Rule 4** — once a token has ≥1 exact match, no typo variants pulled in |
| `drop_tokens_threshold` | `1` | **Rule 3** — drop tokens only when no document has all words |
| `split_join_tokens` | `"fallback"` | **Rule 5** — split/join only when exact match returns nothing |

### Verified behavior
- `camera` → `["3"]` (exact), `camrea` → `["3"]` (2 typos), `logitech` → `["1"]` (exact brand), `logiteck` → `[]` (brand has 0 typos)
- `USB` → `["5"]` (3 letters, exact), `USZ` → `[]` (3 letters, no typo), `wifi` → `["9"]`, `wifiz` → `["9"]` (4 letters, 1 typo)
- `wireless mouse` → `["1"]` (perfect match, no dropping), `anker mouse` → `["6","5"]` (no perfect match, drops `mouse`), `anker cable` → `["5"]` only (perfect match)
- `basket ball` → `["7"]` (join), `waterbottle` → `["8"]` (split), `wifirouter` → `["9"]` (split)

### Output format
`python3 search.py --q "<query>"` prints a single JSON array of matching `id` strings to stdout (e.g. `["3"]` or `[]`), with no other output.
