Typesense exposes a bulk `import` endpoint that accepts newline-delimited JSON (JSONL). Real-world source data is messy: some records have field values that Typesense can automatically coerce, some are broken in ways that can be repaired with well-defined business rules, and some are simply unusable. Your job is to import a large dataset robustly, recover the documents that can be salvaged, and produce an accurate ingestion report. A key subtlety of the Typesense import endpoint is that it always returns `HTTP 200` even when individual documents fail. The response body is JSONL with **one result line per input document, in the same order** as the request. A successful line looks like `{"success": true}` and a failed line looks like `{"success": false, "error": "...", "document": "..."}`. ## How it works

1. **Server bootstrap** — `ensure_server()` checks `GET /health`; if Typesense isn't reachable it launches `/usr/local/bin/typesense-server` against `/tmp/typesense-data` with API key `xyz`, then waits for `/health` to return `{"ok":true}`.
2. **Strict schema** — `recreate_collection()` drops and recreates `catalog` with all five required fields explicitly typed (`sku:string`, `name:string`, `price:float`, `quantity:int32`, `category:string`), ensuring idempotent runs.
3. **First bulk import** — `bulk_import()` POSTs the JSONL to `/collections/catalog/documents/import?action=upsert&dirty_values=coerce_or_reject`, parses the JSONL response, and zips it with the source docs by index (response order matches input order). Coercible numeric strings succeed; bad values surface as per-document failures.
4. **Repair + selective retry** — `repair()` only applies the two permitted rules:
   - Currency-formatted `price` (e.g. `"$1,299.00"` → `1299.0` via a regex that strips anything other than digits, dot, or minus).
   - Missing `category` → set to `"uncategorized"`.
   Anything else (e.g. missing `sku`/`name`, `quantity: "many"`) is left untouched and stays failed.
   Only the docs that were actually repaired are re-imported (also via `action=upsert` with `dirty_values=coerce_or_reject`); docs that still fail after repair are kept in the failed bucket and never re-sent.
5. **Report** — `report.json` is written with exactly the required keys, all counts internally consistent, IDs sorted ascending.
6. **Sanity check** — the script asserts `imported_first_pass + recovered + failed == total` and that `GET /collections/catalog`'s `num_documents` equals `imported_first_pass + recovered`.
