### Collection design

**`likes`** schema is the critical piece — both reference fields carry `async_reference: true`:

```json
{
  "name": "likes",
  "fields": [
    { "name": "user_id",    "type": "string",
      "reference": "users.id",    "async_reference": true },
    { "name": "product_id", "type": "string",
      "reference": "products.id", "async_reference": true }
  ]
}
```

### Async resolution demo

`setup.py` indexes likes `l1`, `l2`, `l3` **before** `users` or `products` exist. Typesense stores `*_sequence_id` resolution pointers (visible in the `.ref` field of each document) and resolves the joins automatically once the referenced documents are indexed later — no manual re-indexing needed.

### Query CLI

```bash
# Which users liked Widget A (p1)?
python3 query.py --product p1
# → ["alice", "bob"]

# Which products did alice (u1) like?
python3 query.py --user u1
# → ["Widget A", "Widget B"]

# Non-existent IDs return empty array
python3 query.py --product p999
# → []
```

The CLI uses Typesense's `$CollectionName(...)` cross-collection JOIN syntax inside `filter_by` and `include_fields`, queries the live server on every call, and handles pagination for large result sets.
