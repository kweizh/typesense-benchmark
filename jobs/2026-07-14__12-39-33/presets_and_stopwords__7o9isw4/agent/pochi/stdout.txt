## What was built

**`run.py`** — a single-file CLI with three modes:

| Command | Behaviour |
|---|---|
| `python3 run.py --setup` | Starts the Typesense server (if not already running), creates the `library` collection, upserts all 5 documents, registers the `en_stopwords` stopwords set, and creates the `library_default` preset. Fully idempotent. |
| `python3 run.py --q "<query>"` | Searches using only `q` + `preset=library_default`; no extra parameters sent. |
| `python3 run.py --q "<query>" --explicit` | Searches with `query_by=title,author`, `sort_by=points:desc`, `stopwords=en_stopwords` — no preset reference. |

Both search paths emit `{"found": <int>, "hits": ["<id>", …]}`.

## Verified results

```

$ python3 run.py --q "the"                 # pure stopword → no results
{"found": 0, "hits": []}
```

## Key design notes

* **Stopwords** (`the`, `a`, `of`, `and`) are stripped from the *query* at search time; they remain in indexed document titles.
* The **preset value** is stored as a flat object (not a `searches` array) so it works correctly with the single-collection `/documents/search` endpoint.
* **Idempotency**: collection creation checks for 200 before skipping; documents use `action=upsert`; stopwords/presets check for existence via GET before issuing PUT.
* Uses only Python stdlib (`urllib`, `subprocess`, `json`, `argparse`) — no third-party packages required.
