### `build_index.py`

* **Idempotent**: drops the `airports` collection if it already exists, then recreates it fresh every run.
* **Schema**: collection has `id`, `name`, `iata`, `city`, `country`, and a `location` field of type `geopoint`.
* **geopoint order**: each record's `location` is set to `[lat, lng]` (latitude first â€” the order Typesense requires).
* **Bulk import**: all records are assembled into a JSONL string and sent via the `import_` endpoint in a single call. Any per-document errors are surfaced to stderr.
* Verified: 20 records imported on first run; re-run drops+recreates cleanly with 20 records again.

### `search.py`

* Accepts `--lat`, `--lng`, and `--radius-km` via `argparse`.
* **Radius filter**: `filter_by = "location:(lat, lng, R km)"` â€” excludes airports outside the radius at the Typesense level.
* **Distance sort**: `sort_by = "location(lat, lng):asc"` â€” Typesense returns hits in ascending geo distance order.
* **Distance value**: read directly from `hit["geo_distance_meters"]["location"]` â€” no client-side recomputation.
* **Output schema** (exact keys, integer distance):
  ```json
  {
    "reference": {"lat": ..., "lng": ..., "radius_km": ...},
    "found": <int equal to len(results)>,
    "results": [{"id": "...", "iata": "...", "name": "...", "distance_meters": <int>}, ...]
  }
  ```
