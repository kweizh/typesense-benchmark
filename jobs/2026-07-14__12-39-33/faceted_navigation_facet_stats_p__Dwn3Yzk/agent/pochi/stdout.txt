## `/home/user/facet-nav/facet_nav.py`

### Request â†’ Response flow

**1. Filter building (`build_filter_by`)**  
Converts the `filters` dict into a Typesense `filter_by` string.  Multi-value categorical fields become `field:[v1,v2]` (OR semantics). Price becomes a range expression (`field:[lo..hi]`).  An `exclude_field` parameter omits one field's filter clause â€” this is the key mechanism for disjunctive faceting.

**2. Main query (all filters active)**  
Sent with every active filter applied. Provides:
- `found` â€” documents matching every filter + `q`.
- `price_stats` â€” min/max/avg/sum from Typesense's facet `stats` object on the `price` field, rounded to 2 decimal places (zeros when `found == 0`).
- Baseline facet counts for any categorical field that has **no** active filter (those fields' own counts are unaffected by disjunctive semantics).

**3. Disjunctive queries (one per active categorical filter)**  
For each of `brand`, `category`, `tags` that has a selected value, a separate Typesense search is issued with that field's own filter **removed** but all other filters kept. The resulting counts for that field show what a shopper would see after toggling off their current selection â€” exactly the multi-select UX semantics required.

**4. Optional `facet_query` request**  
When `facet_query` is supplied, a further search is issued using Typesense's `facet_query` parameter (`field:prefix`) against the disjunctive filter for that field. Results appear in `facet_query_matches` sorted by count descending.
