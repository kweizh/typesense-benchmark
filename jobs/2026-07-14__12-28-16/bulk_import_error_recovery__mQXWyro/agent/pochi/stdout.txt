## What it does

1. **Ensures a Typesense server is running** at `http://localhost:8108` with API key `xyz`. It health-checks first and only starts `/usr/local/bin/typesense-server` (against `/tmp/typesense-data`) if one isn't already reachable.
2. **Recreates the `catalog` collection** with a strict, explicitly-typed schema (`sku`/`name`/`category` as `string`, `price` as `float`, `quantity` as `int32`, `id` as the document id) — making the pipeline idempotent.
3. **Bulk-imports the dataset** using the JSONL import endpoint with `dirty_values=coerce_or_reject` and `action=upsert`. Crucially, it does **not** trust the HTTP 200 status; it parses the per-document JSONL response (whose order matches the input order) and maps each result back to its source record to find failures.
4. **Applies the two allowed repair rules** to the failed documents:
   - Currency-formatted price (e.g. `"$1,299.00"`) → normalized to a float.
   - Missing `category` → set to `"uncategorized"`.
5. **Re-imports only the repaired documents.** Documents with no applicable repair (missing `sku`, missing `name`, non-numeric `quantity`) are never re-imported and remain failed.
6. **Writes `report.json`** with exactly the required keys and verifies consistency against the live collection count.
