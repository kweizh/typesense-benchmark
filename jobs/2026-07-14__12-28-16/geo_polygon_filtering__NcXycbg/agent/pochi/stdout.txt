## Files Created

### `start.sh`
Launches the Typesense server (native binary) on `localhost:8108` using the `TYPESENSE_API_KEY` environment variable. It starts the server in the background with `nohup`, polls `http://localhost:8108/health` until it reports `"ok":true` (up to 60 s), then returns. It is safe to call when the server is already running — it checks the health endpoint first and exits early if healthy.

### `seed.py`
Idempotent seeding script. Connects to Typesense, deletes the `hubs` collection if it exists, recreates it with the schema:
- `name`: string
- `status`: string, facetable
- `location`: geopoint

Then imports all 10 hub documents with locations stored as `[latitude, longitude]`.

### `search.py`
CLI that accepts `--polygon` (comma-separated alternating lat/lng values) and optional `--exclude-status`. It builds a Typesense `filter_by` expression using the polygon syntax `location:(lat1,lng1, lat2,lng2, ...)` and optionally appends `&& status:!=<value>` for status exclusion. Prints `{"hub_ids": [...]}` sorted in ascending lexicographic order, or `{"hub_ids": []}` when nothing matches.
